# Challenge 2 — Write Your PRD

[< Previous Challenge](challenge-01.md) - **[Home](../Readme.md)** - [Next Challenge >](challenge-03.md)

**Duration:** 20 minutes

## Goal

Write a Product Requirements Document (PRD) for the **Task Board** app. This is the only step where you write anything substantial — everything downstream (FRDs, tests, code, deployment) traces back to it.

## Actions

* Open `specs/prd.md` in VS Code and replace its contents with your own product spec
* Include the following sections:
    * **What the app does** — one or two sentences describing the product
    * **Who uses it** — a single user, no auth
    * **Key user stories** (at least 3–5) — what can the user do, and what does success look like for each?
    * **Technical constraints** — in-memory store, no database, no auth
* Write specific acceptance criteria for each user story (e.g., "The task appears in the To Do column" not "the task is created")
* Explicitly list what is **out of scope** (non-goals)

> 💡 A sample PRD is available at `docs/microhack-sample-prd.md` in your repo for reference.

## Success Criteria

* `specs/prd.md` contains **your own** PRD (not the template placeholder)
* At least 3–5 user stories with acceptance criteria are present
* Technical constraints (in-memory, no auth, no database) are specified
* Non-goals are listed
* The file is saved
